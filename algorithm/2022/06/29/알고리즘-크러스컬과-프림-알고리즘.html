#Algorithm#C++

탐욕법

최적화 문제를 해결하기 위한 방법으로, 최적화 문제란 가능한 모든 대안 중에서 가장 좋은 최적해를 고르는 문제이다. 전체적인 흐름에서의 최적으로 결정하는 것이 아닌 현재 최적이라고 판단되는 결정을 한다.

이 방법을 사용하기 위해서는 최적 부분구조(optimal substructure)를 만족해야하며, 이는 주어 문제의 최적해가 부문제들의 최적해로부터 효율적으로 생성되어야 한다는 뜻이다. 이때의 결정은 취소하거나 변경하지 않기 때문에 빠르다.

아래 나오는 2개의 알고리즘 (크러스컬, 프림)은 모두 최소 신장트리 (MST : Minimum Spanning Tree) 구하기에 사용하는 대표적인 알고리즘이다. 이 신장 트리란 연결 그래프에서 사이클을 형성하는 간선을 제거하여 만든 트리이다. 아래 그림에서 1)는 사이클이 존재하는 그래프이고, 2)는 사이클이 존재하지 않는 신장트리이다.

이 알고리즘은 “최소한의 비용으로 모든 도시를 연결하는 도로망을 건설하라.” 와 같은 문제에 있을때 유용하게 사용한다.

크러스컬 알고리즘

다른 건 생각하지 않고 각 단계에서 가중치가 가장 작은 간선부터 선택하며, 이때 사이클이 만들어진다면 그 간선은 버리게 된다. 이를 위한 순서는 아래와 같다.

  1. 선택 과정 전에 모든 간선을 오름차순으로 정렬해두어 선택 시간을 줄인다.

  2. 각 정점을 상호배타적인 하나의 집합으로 만든다.

이때 위는 “번호”, “아래는 집합 번호”라고 생각하며, 이는 트리형태를 1차원으로 표현한 것이다.

  1. __“Union Find”__연산을 사용하여 사이클을 형성하지 않는다면 두 집합을 하나로 합친다. 이 연산은 아래에서 추가로 설명한다.

    만약 1과 2를 판단한다면 우선 먼저, 각 1과 2의 집합 번호를 Find 연산을 활용하여 알아내고, 만약 다르다면 사이클이 형성되지 않는다고 판단하고, Union 연산을 진행하여 합친다. 그럼 아래와 같은 결과가 출력된다. 이는 1과 2가 같은 집합에 포함되어 있다는 것을 의미한다.

  2. 3번의 과정을 반복한다.

Union & Find

이는 배타적 집합의 개념을 이용하며, 총 3가지 연산으로 구분된다. 최초에 정점 하나를 원소로 하는 집합을 만드는 “init_set” 연산, 임의의 정점이 어느 집합의 원소인지 알아내는 “find” 연산, 두 집합을 하나로 합치는 “union” 연산이 필요하다. 코드는 다음과 같다.

```c++
// Union/Find 코드
int Find(int v) {
  if (v == arr[v]) return v;
  else return arr[v] = Find(arr[v]);
}

void Union(int a, int b) {
  a = Find(a);
  b = Find(b);
  if (a != b) arr[a] = b;
}
```

프림 알고리즘

가중치가 가장 낮은 간선부터 선택하는 점은 크러스컬 알고리즘과 동일하지만, 트리를 처음부터 끝까지 유지한다는 점이다. 즉 실행 내내 트리는 하나만 존재해야한다. 이와 비슷한 알고리즘은 다익스트라 알고리즘이 존재한다.

  1. 아무 정점이나 선택하여 시작 트리로 초기화한다.
  2. 매 단계에서 간선 하나와 정점하나를 붙여나가면서 트리를 유지한다.
  3. 모든 정점을 선택하면 종료한다.

이때 성능을 판단하는 것은 최소 가중치의 간선을 선택하는 방법인데, 각 정점에 대해서 distance(v)와 neareset(v)를 정의한다. distance는 갈 수 있는 가장 작은 가중치를 의미하며, nearest는 상대 정점을 의미한다.
