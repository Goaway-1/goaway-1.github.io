객체지향프로그래밍

객체 지향 프로그래밍 (Object-Oriented Programming, OOP)란 어떤 프로그램의 일부분에 해당하는 작은 부품, 즉 객체를 먼저 만들고 이렇게 만들어진 여러 객체들을 조립해서 하나의 완성된 프로그램을 만드는 프로그래밍 방법론이다. 마치 자동차를 만들때, 여러 부품들로 나누어 개발하고, 합쳐 하나의 자동차를 만드는 것과 같다.

※ 절차 지향 프로그래밍 : 실행하고자 하는 절차대로 일련의 명령어를 나열한 프로그래밍

객체란 객체 지향 프로그래밍의 가장 기본적인 단위로 모든 사건은 객체들간의 상호작용을 통해서 발생한다. 이를 추상화하여 ‘속성’과 ‘기능’으로 분류한 후에 ‘변수’와 ‘함수’로 정의한다.

특징

가장 큰 특징은 “추상화, 상속, 다형성, 캡슐화”로 4가지로 구성된다.

먼저 “추상화(Abstraction)”는 실제로 객체화하는 것이 아닌 필요한 정보만을 중심으로 간소화하는 것을 의미한다. 즉, 추상 클래스에서는 어떤 객체가 수행해야하는 역할만을 규정해두고, 실질적인 구현은 해당 클래스를 상속받은 객체에서 구현하도록 프로그램을 설계하는 것을 의미한다.

“상속(Ingeritance)”이란 상위 클래스의 기능을 하위 클래스가 사용할 수 있는 개념이다. 이는 여러 클래스에서 사용되는 기능을 하나의 클래스로 분리해서 사용할 수 있어 중복을 제거할 수 있다.

“다형성(Polymorphism)”이란 객체간의 관계를 유연하게 해주는 것이 특징으로 어떤 객체의 속성이나 기능이 상황에 따라 여러가지 형태를 가질 수 있다는 뜻이다. 상속 관계에 있는 부모 클래스에서 이미 정의된 함수를 자식 클래스에서 다시 재정의(Overriding)할 수 있다.

“캡술화(Encapsulation)”란 클래스 안에 서로 연관되어 있는 속성과 기능들을 하나의 캡슐로 만들어 데이터를 외부로부터 보호하는 것을 의미한다. 이를 통해서 외부로 부터 클래스에 정의된 속성과 기능들을 보호할 수 있고, 내부의 동작을 감추고 필요한 부분만 외부에 노출할 수 있다. 접근제어자 (public, privat. protected)를 통해서 제어한다.

  • ※ 오버로딩(Overloading) : 같은 이름의 메서드 여러개를 가지면서 매개변수의 유형과 개수를 다르게하는 기술
  • ※ 오버라이딩(Overriding) : 상위 클래스가 가지고 있는 함수를 자식 클래스에서 재정의하여 사용하는 기술

장점

  • 재사용성 : 상속을 통해서 코드의 재사용을 높일 수 있음
  • 생산성 향상 : 객체는 독립적이기 때문에 개발의 생산성을 향상시킬 수 있음
  • 자연스러운 모델링 : 실생활의 구조를 녹인 것이기에 자연스로움

단점

  • 개발/실행속도 늦음

C++

기존 C언어에 여러 가지 기능을 추가하여 만든 프로그래밍 언어로 ‘절차 지향적 언어’의 특징과 ‘객체 지향적 언어’를 동시에 사용 가능하다.

C++ 통합 개발 환경 (IDE - Integrated Development Environment)이란 소스코드 작성, 컴파일, 디버깅 기능을 가지고 있는 것입니다. C나 C++의 통합개발환경으로는 비주얼 스튜디오, 비쥬얼 스튜디오 코드, devcpp 등이 있습니다.

특징

  • C언어의 문법적 체계를 계승했기에 C언어로 작성된 프로그램과의 호환성을 갖는다.
  • 객체 지향 개념을 도입하여 재사용과 생산성, 캡슐화, 상속, 다형성등의 특징을 지닌다.
  • 타입 체크를 엄격히 하기에 실행 시간 오류의 가능성을 줄여 디버깅을 돕는다.
  • 객체 지향 개념의 도입으로 함수의 호출이 잦아기면 비효율적인데, 이를 막기 위해서 “인라인 함수”등을 사용한 함수 호출로 실행 시간의 효율성 저하를 최소화한다.
    • 인라인 함수, 함수 중복(Overloading), 디폴트 매개 변수, 참조, 참조에 의한 호출, new&delete, 재정의(Overriding), 제너릭 함수와 클래스
    • lnline : 재귀, static, 반복문, switch, goto는 허용되지 않는다.
  • 캡슐화 : 캡슐 역할을 하는 것이 클래스(class)이며, 이를 통해 생겨난 것은 실체(instance)이다.
  • 상속성 : 자식이 부모로부터 멤버를 받아 태어나며, 코드의 재사용성을 높여 생산성을 향상
  • 다형성 : 하나의 기능이 경우에 따라 다르게 작동하는 현상

링킹(linking)

컴파일 후 기계어로 구성된 목적 파일이 생성되어도 목적 파일은 바로 실행될 수 없다. 개발자가 작성한 C++이 다른 라이브러리등을 이용하는 경우 참조만 하고, 포함하지 않기 때문에 실행되지 않는다.

즉, 목적 파일(object file)은 온전한 실행 파일이 아니며, 이들과 결합되어야만 비로소 실행 가능한 파일이 된다. 여기서 “링킹(linginh)”이란 C++ 표준 라이브러리나 다른 파일 속에 있는 함수, 객체, 데이터를 포함하여 실행에 필요한 모든 기계어 코드를 확보하여 하나의 실행 파일로 만드는 과정이다. 이 결과 하나의 실행 파일(.exe)이 만들어진다.

얕은 복사 vs 깊은 복사

복사 함수를 따로 지정해주지 않으면, 얕은 복사를 실행하곤 하는데 이때 만약 변수가 포인터 변수일때는 해당 변수의 포인터 값을 그래도 복사하게 된다. 이렇게 되면 A객체를 복사한 B객체의 포인터 변수를 수정할때, A객체의 포인터 변수 또한 수정되게 된다.

이를 해결하는 것이 깊은 복사로 포인터 변수에 대해 새로운 주소를 할당하여, 오류를 막을 수 있다.

가상 함수

부모 클래스의 함수를 자식 클래스에서 재정의(오버라이딩)하기 위해서 만드는 함수이다. ‘virtual’이라는 키워드를 사용하며. 다형성을 구현할때 사용하며, 반드시 재정의를 해야하는 것은 아니다.

이외에 추상 클래스도 존재하는데, 이는 내용은 구현하지 않으며, 인스턴스화 할 수 없다.

namespace와 std::

개발자를 구분하거나 다른 사람이 개발한 소스/목적 파일을 가져와 사용하는 경우 이름에 대한 충돌이 발생할 수 있다. 이를 구분하기 위한 것이 이름공간 (namespace)이다.

std란 C++ 표준 라이브러리에서 선언된 이름을 사용할때, 이를 접두어로 붙어야한다. 예를 들면 표준 입출력인 cout, cin, endl등이 있다. 이는 매우 번거롭기 때문에 “using” 지시어를 사용하여 이름 공간 접두어를 생략한다.

using std::cout;        // cout에 대해서만 std:: 생략
using namespace std;    // std 이름 공간에 선언된 모든 이름에 std:: 생략
  • ※ 범위 지정 연산자 -> ::

구조체

C언어와의 호환성을 위해 구조체(Struct)을 지원하며, 클래스와 동일한 구조와 기능적으로 동일하다. 딱 하나 다른 점은 “디폴트 접근 지정”이 클래스는 private이지만, 구조체는 public이다.

멤버 변수만 있는 자료 구조의 경우 구조체를 사용하기도 하지만, 굳이 C언어의 자산을 사용할 필요는 없다.

포인터

포인터를 이용한 객체 멤버 근하기 위해서 점(.) 연산자를 사용하지만, 객체 포인터로 멤버에 접근할때는 ‘->’연산자를 사용한다.

d.getArea();
d->getArea();
(*d).getArea();
