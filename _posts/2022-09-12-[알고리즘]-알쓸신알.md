# Union 공유체

c++에는 묶음 타입으로 struct, class, union 이렇게 세가지가 존재한다. 클래스와 구조체는 많이 사용하지만 구조체(union)는 들어본적이 거의 없을 것이다. 이는 무엇이고 구조체와는 무엇이 다른지에 대해 설명하고자 한다.

공유체는 <ins>__"하나의 메모리 저장소를 여러개의 데이터 타입을 가진 멤버들이 공유 하는것"이라고__</ins> 정의한다. 즉 하나의 멤버 변수에 값을 넣었다면, 다른 멤버 변수를 호출해도 공유된 메모리 저장소에 저장된 값이 호출된다.

구조체의 경우 각각의 멤버 변수마다 메모리를 알맞게 할당하지만, 공유체는 하나의 메모리에 여러 변수가 할당된다고 생각하면된다. 이 때문에 해당 변수의 바이트 값이 다르게 되면 에러가 발생한다.

```cpp
union Data1 {
	int		v_int;
	float	v_float;
	char	v_string[4];
};
```

위 코드에서는 Data1를 공유체로 선언하였다. 이 공유체의 <ins>__멤버변수들은 모두 크기가 4byte이기 때문에, 공유체의 총 크기는 4byte가 된다.__</ins> 만약 이를 구조체로 바꾼다면 12byte가 될것이다.

# 이건 뭐노

빈번히 발생하는 연산은 "스택공간"에서 데이터를 처리하는 데 중점을 두는 struct를 사용한다.
구조체의 연산을 선언할 때는 가급적 함수 호출 없이 빠르게 진행될 수 있도록 인라인 키워드를 사용한다.
컴파일 과정에서도 미리 계산될 수 있도록 constexpr 사용
