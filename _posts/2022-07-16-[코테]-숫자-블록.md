## 문제

  그렙시에는 0으로 된 도로에 숫자 블록을 설치하기로 하였습니다. 숫자 블록의 규칙은 다음과 같습니다.

  블록의 번호가 n 일 때, 가장 처음 블록은 n * 2번째 위치에 설치합니다. 그다음은 n * 3, 그다음은 n * 4, ...로 진행합니다.만약 기존에 블록이 깔려있는 자리라면 그 블록을빼고 새로운 블록으로 집어넣습니다.

  예를 들어 1번 블록은 2,3,4,5, ... 인 위치에 우선 설치합니다. 그다음 2번 블록은 4,6,8,10, ... 인 위치에 설치하고, 3번 블록은 6,9,12... 인 위치에 설치합니다.

  이렇게 3번 블록까지 설치하고 나면 첫 10개의 블록은 0, 1, 1, 2, 1, 3, 1, 2, 3, 2이됩니다.

  그렙시는 길이가 1,000,000,000인 도로에 1번 블록부터 시작하여 10,000,000번 블록까지 위의 규칙으로 모두 놓았습니다.

  그렙시의 시장님은 특정 구간의 어떤 블록이 깔려 있는지 알고 싶습니다.

  구간을 나타내는 두 수 begin, end 가 매개변수로 주어 질 때, 그 구간에 깔려 있는 블록의 숫자 배열(리스트)을 return하는 solution 함수를 완성해 주세요.

## 조건

  - begin, end 는 1 이상 1,000,000,000이하의 자연수 이고, begin는 항상 end보다 작습니다.
  - end - begin 의 값은 항상 10,000을 넘지 않습니다.

## 입출력 예시

  |begin|end|result|
  |:--:|:--:|:--:|
  |1|10|[0, 1, 1, 2, 1, 3, 1, 4, 3, 5]|

## 풀이방식

### 1번 풀이방식 

  > 효율성 만족 X

  end-begin의 값이 항상 10000을 넘지 않는다고 하여 "시간초과를 신경써도 되지 않겠다"라고 생각했는데, 정확성은 통과했으나 효율성을 통과하지 못했다.

  이는 각 블록에 맞는 것을 while구문을 통해서 찾은 것이다.

  ```cpp
  vector<int> solution(long long begin, long long end) {
    vector<int> answer((end - begin) + 1);
    int index = end - begin;

    for (int i = end; i >= begin; i--) {
      // if i is 1. than answer is 0
      if (i == 1) {
        answer[index--] = 0;
        break;
      }

      //get
      int cnt = i / 2;
      while (i % cnt != 0) cnt--;

      //set
      answer[index--] = cnt;
    }
    return answer;
  }
  ```

### 2번 풀이방식 

  > 효율성 만족 X

  단순히 특정 숫자(num)부터 2,3,4,....으로 딱 나누어 떨어질때, 블록의 번호에 num을 할당한다.

  특정 숫자는 하나씩 크기를 줄여가며 진행한다. 즉 end부터 계산하는 것이 아닌 __블록의 번호에 들어갈 값을 기준으로 계산하는 방법이다.__

  이는 시간적으로 효율적으로 구성했으나, 예외 처리하기 위해 사용한 "while(begin > target) target = num * (++tmp);" 에서 많은 시간이 소요되었다.
 
  ```cpp
  vector<int> solution(long long begin, long long end) {
    vector<int> answer((end - begin) + 1,1);

    long long num = end / 2;

    if(begin == 1) answer[0] = 0;
    while (num > 1) {
      long long tmp = 2;
      long long target = num * tmp;
        
      while(begin > target) target = num * (++tmp);
      while (target <= end && begin <= target) {
        if (answer[target - begin] == 1) answer[target - begin] = num;
        target = num * (++tmp);
      }
      num--;
    }
    return answer;
  }
  ```

### 3번 풀이방식 

  > 효율성 만족 O

  2번 풀이방식보다 느리지만, 효율성은 통과한 답안이다.

  이는 정말 단순하게 블록의 번호에 맞는 숫자를 하나씩 찾는 방법으로, 블록의 번호는 최소 본인 숫자 루트까지 나올 수 있기 때문에 __"i < sqrt(number) + 1"__ 조건을 가지고 진행한다.

  ```c++
  long long cal(long long number) {
    if (number == 1) return 0;
    for (long long i = 2; i < sqrt(number) + 1; i++) {
      long long val = number / i;
      if (val > 10000000) continue;
      if (number % i == 0) return val;
    }
    return 1;
  }

  vector<int> solution(long long begin, long long end) {
    vector<int> answer;
    for (long long i = begin; i <= end; i++) {
      long long val = cal(i);
      answer.push_back((int)val);
    }
    return answer;
  }
  ```
