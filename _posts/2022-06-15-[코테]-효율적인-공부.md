## 문제

  표는 N(1<=N<=1,000,000)시간의 일정을 겹쳐지는 M(1<=M<=1,000)구간별로 공부의 효율성이 표시되어 있다. 각 구간은 시작시간(0 <= st < N)과 끝나는 시간(st < et <= N) 그리고 해당 구간에서의 공부의 효율성이 주어진다. 철수는 한 구간을 공부하고 나면 꼭 휴식시간(1 <= R <= N)을 가져야만 합니다. 철수가 N시간동안 공부를 할 때 각 구간을 잘 선택해서 공부를 열심히 한다면 가장 높은 효율 성을 얼마인지 출력하는 프로그램을 작성하세요.

## 조건

  - 첫째 줄은 세 개의 정수 N(총 시간), M(구간의 개수), R(휴식시간)이 주어진다.
  - 둘째 줄부터 M+1줄 까지는 각 구간의 시작시간 끝나는 시간 그리고 해당구간의 공부효율성이 주어진다.

## 풀이방식

  먼저 공부는 시간의 흐름에 영향을 받기 때문에 Time이라는 구조체를 만들고, 이 구조체의 정렬 기준을 Et(종료 시간)이 가장 작은 것부터 내림차순으로 정렬한다.

  정렬 후 각 __공부 구간에 대하여 얻을 수 있는 효율성을 dy배열에 저장한다.__ 이는 뒤에 있는 구간의 효율성에 영향을 받으며, 즉 1구역의 효율성의 합은 2구역의 효율성의 합에 영향을 준다.

  N구역은 N-1 ~ 1구역까지에 대해서 비교를 진행하여 효율성을 구하는데 조건은 다음과 같으며, 조건을 만족하는 경우 해당 구역의 dy[i]를 갱신한다.

  |__조건__|__설명__|
  |:--:|:--:|
  |1. N-i 끝나는 시간 + 쉬는 시간 <= N구역의 시작 시간|공부 후 쉬는 시간이 꼭 필요하기 때문에 현재의 구역을 중심으로 진행될 수 있는지 체크|
  |2. dy[j]의 효율성 + 현재 구역의 효율성 > dy[i] 효율성 |현재의 효율성과 j구역의 효율성을 더했을때 변경된 효율성보다 큰지 체크|

  이 로직은 앞에서 뒤의 흐름으로 지나가기 때문에 앞에서 계산한 정보를 다시 이용할 수 있다.

    ```cpp
    #include <iostream>
    #include <vector>
    #include <algorithm>

    struct time{
      int St, Et, val;
      
      time(int a,int b, int c){
        St = a;
        Et = b;
        val = c;
      }
      
      bool operator <(const time &b) const{
        return Et < b.Et;
      }
    };

    int main(){
      int n,m,r,answer=INT_MIN;
      cin >> n >> m >> r;
      vector<time> arr;
      vector<int> dy(m+1, 0);
      
      // input
      int tmp[3];
      for(int i = 0;i < m;i++) {
        cin >> tmp[0] >> tmp[1] >> tmp[2];
        arr.push_back(time(tmp[0],tmp[1],tmp[2]));
      }
      
      sort(arr.begin(),arr.end());
      
      // Calculation
      for(int i = 0;i < m;i++) {
        dy[i] = arr[i].val;
        for(int j=i-1;j>=0;j--){
          if(arr[j].Et + r <= arr[i].St && dy[j] + arr[i].val > dy[i]){
            dy[i] = dy[j] + arr[i].val;
          }
        }
        answer = max(answer, dy[i]);
      }
      cout << answer;
      return 0;
    }
    ```
