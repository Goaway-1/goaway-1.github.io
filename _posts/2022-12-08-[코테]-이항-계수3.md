## 문제

  자연수 N과 정수 K가 주어졌을 때 이항 계수 (N K)를 1,000,000,007로 나눈 나머지를 구하는 프로그램을 작성하시오.
  
  [링크](https://www.acmicpc.net/problem/11401)
  
## 조건

  - 첫째 줄에 \(N\)과 \(K\)가 주어진다. (1 ≤ \(N\) ≤ 4,000,000, 0 ≤ \(K\) ≤ \(N\))

## 입출력 예시
  
  |N|K|result|
  |:--:|:--:|:--:|
  |5|2|10|

## 풀이방식

### 나의 풀이방식 (오답)

  > 가장 큰 수인 n!의 값을 구하고, 구하는 과정에서 각 팩토리얼 값을 배열에 저장해둔 후 계산식에서 사용

  식 : <ins>__n! / (k! * (n - k)!)__</ins>

  이때 모든 값이 너무 커질 수 있기 때문에 "1,000,000,007"로 나누어주었는데 안되었다. 이는 내가 모르는 방법이 있다고 판단했다.

  ```cpp
  unsigned long long val[4000001];

  int main() {
    unsigned long long n = 0, k = 0, answer = 0;
    cin >> n >> k;
    if (n == k || n == 0) return 0;

    val[1] = 1;
    for (unsigned long long i = 2; i <= n; i++) val[i] = (val[i - 1] * i) % 1000000007;

    answer = ((val[n] / val[n - k]) / val[k]) % 1000000007;

    cout << answer;
  }
  ```

### 다른이의 풀이방식 (정답)

  > 페르마의 소정리를 활용하여, 모듈로 연산이 가능하도록 수정

  이전 나의 풀이에서 문제점은 모듈로 연산을 사용할 수 없는 환경(나누셈)에서 모듈로 연산을 사용했기 때문이다.

  이를 해결하기 위해서는 <ins>"페르마의 소정리"</ins> 개념을 사용하여 "A * B"형식으로 만들어 주어야하며, 아래의 식과 같다.

  ```
  ※ MOD = 1,000,000,007
  (A / B) % MOD --> (A % MOD) * ((B ^ -1) % MOD)
  ```

  그렇기에 계산 식은 아래 그림과 같이 변경된다.
  <img src="https://raw.githubusercontent.com/Goaway-1/goaway-1.github.io/master/_posts/images/Number_Theory.png" height="350" title="Number_Theory">

  ```cpp
  #include <iostream>
  #define MOD 1000000007

  long long solve(long long &n, long long x) {
    long long ans = 1;

    if (x == 0) return 1;

    if (x % 2 == 1) return n * solve(n, x - 1) % MOD;
    else
    {
      // 조금 더 빠르게 진행하기 위해서...
      ans = solve(n, x / 2);
      return (ans * ans) % MOD;
    }
  }

  int main() {
    int n, k;
    cin >> n >> k;

    // A = (n - k)!
    // B = K!    
    long long A = 1, B = 1;

    /** A, B의 팩토리얼 계산 */
    for (int i = n; i >= n - k + 1; --i) A = (A * i) % MOD;
    for (int i = 2; i <= k; ++i) B = (B * i) % MOD;

    // K!^(mod-2) 
    // 이때 분할정복을 이용하여 계산
    cout << (A * solve(B, MOD - 2)) % MOD;

    return 0;
  }
  ```


## 개선점
  - __모듈로 연산__ : 어떤 한 숫자를 다른 숫자로 나눈 나머지를 구하는 연산으로, 나머지 연산이라고도 함. 덧셈/뺄셈/곱셈에만 적용 가능
