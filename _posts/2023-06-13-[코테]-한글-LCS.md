## 문제

  LCS(Longest Common Subsequence, 최장 공통 부분 수열)문제는 두 수열이 주어졌을 때, 모두의 부분 수열이 되는 수열 중 가장 긴 것을 찾는 문제이다.

  예를 들어, "최백준"과 "백준온라인"의 LCS는 "백준"이고, "스타트링크"와 "스트라토캐스터"의 LCS는 "스트"이다.

  [링크](https://www.acmicpc.net/problem/15482)
  
## 조건

  - 첫째 줄과 둘째 줄에 두 문자열이 주어진다. 
    - 문자열은 최대 1000글자이고, 유니코드 U+AC00(가)부터 U+D7A3(힣)까지로만 이루어져 있으며, UTF-8로 인코딩 되어 있다. 
    
## 입출력 예시
  
  |str1|str2|answer|
  |:--:|:--:|:--:|
  |최백준|백준온라인|2|
  |스타트링크|스트라토캐스터|2|
  |선데이코딩|딩코이데선|1|
  |가나다라가나다라|가다나가다라|5|

## 풀이방식

  한글이라니.. 보자마자 당황을 금치 못했으나, 테스트 해보니 한글은 하나의 단어가 2개 인덱스를 잡아먹었다. 그래서 문자를 2개의 인덱스씩 잘라서 비교했하여 LCS와 동일한 방식으로 처리하였다.

  근데 본인 컴퓨터에서는 잘 돌아가는데, 제출하자 마자 '런타임 에러(OutOfBounds)' 가 났다. 범위를 넘어갔나 싶어서 1000글자를 넣어 테스트 해봤는데, 이상이 없었다. 찾아보니 다른 풀이에서는 3개씩 잘라서 했길래 혹시나 해서 3으로 바꿔서 제출하니 성공했다. 다만 이는 본인 컴퓨터에서 안된다.
  
  이유는 뭘까...? 하드웨어? 아니면 컴파일러?

  ```cpp
  /** 백준 정답 */
  #include <iostream>
  #include <string>
  using namespace std;

  int LCS[1001][1001];

  int main()
  {
    ios_base::sync_with_stdio(0);
    cin.tie(0);
    cout.tie(0);

    string str1, str2;
    cin >> str1 >> str2;

    // dp의 최대 인덱스 값
    int n = str1.size() / 3;
    int m = str2.size() / 3;

    // 계산
    for (int i = 1; i <= n; i++)
    {
      for (int j = 1; j <= m; j++)
      {
        if (str1.substr((i * 3) - 3, 3) == str2.substr((j * 3) - 3, 3))
        {
          LCS[i][j] = LCS[i - 1][j - 1] + 1;
        }
        else
        {
          LCS[i][j] = max(LCS[i - 1][j], LCS[i][j - 1]);
        }
      }
    }
    cout << LCS[n][m] << endl;
  } 
  ```

        }
      }
   }

  }
