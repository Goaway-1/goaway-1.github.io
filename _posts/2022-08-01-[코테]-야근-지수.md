## 문제

  회사원 Demi는 가끔은 야근을 하는데요, 야근을 하면 야근 피로도가 쌓입니다. 야근 피로도는 야근을 시작한 시점에서 남은 일의 작업량을 제곱하여 더한 값입니다. Demi는 N시간 동안 야근 피로도를 최소화하도록 일할 겁니다.Demi가 1시간 동안 작업량 1만큼을 처리할 수 있다고 할 때, 퇴근까지 남은 N 시간과 각 일에 대한 작업량 works에 대해 야근 피로도를 최소화한 값을 리턴하는 함수 solution을 완성해주세요.

## 조건

  - works는 길이 1 이상, 20,000 이하인 배열입니다.
  - works의 원소는 50000 이하인 자연수입니다.
  - n은 1,000,000 이하인 자연수입니다.

## 입출력 예시

  |works|n|result|
  |:--:|:--:|:--:|
  |[4, 3, 3]|4|12|
  |[2, 1, 2]|1|6|
  |[1,1]|3|0|
  
## 풀이방식

### 1번 풀이방식 (오답)

  > 최고의 집합처럼 야무지게 수학써서 해보려했으나 야무지게 실패

  그 이유는 일의 작업량을 모두 합산하고 나누었기 때문에 기존 특정 일에 대해 작업량이 증가할 수 있다는 점이다.

  ```cpp
  long long solution(int n, vector<int> works) {
    long long answer = 0;

    long long cnt = (- 1 * n);
    for (int i = 0; i < works.size(); i++)cnt += works[i];

    //완수할 수 있으면
    if(cnt <= 0) return 0;
    else {
      long long WorkCnt = cnt / works.size();       
      long long RemainCnt = cnt % works.size();       //나머지 값

      for (int i = 0; i < works.size(); i++) {
        if (RemainCnt-- > 0) answer += pow(WorkCnt + 1,2);
        else answer += pow(WorkCnt, 2);
      }
    }  
    return answer;
  }
  ```

### 2번 풀이방식 (정답)

  > 야무지게 해결

  가장 큰 숫자부터 작게 만들어야 피로도가 낮아진다. 즉, 큰 숫자가 있다면 피로도가 증가하고, 다른 작업량과 비슷해야만 최적의 피로도를 구할 수있다. 그렇기 때문에  works배열을 큰 순서에 맞게 정렬한다.

  1. works 배열을 내림차순으로 정렬
  2. result가 0이 되는 경우를 계산하기 위해서, 모든 수를 더하고 n으로 뺏을 때 0보다 작은지 판단 
  3. 맨 앞 index와 다음 index의 값을 비교하고, 둘의 값이 차이가 있다면 그 차이만큼 빼고 n도 제거
      - 그 값들의 크기가 같을때까지 빼는데, 이때 n이 남는지 판단하여 남는다면 그냥 진행하고, 그렇지 않다면 "0 ~ index"까지 돌면서 하나씩 빼준다.
      - 만약 값들의 크기가 같고 마지막 index라면, n이 0이 될때까지 "0 ~ index"까지 돌면서 하나씩 빼준다.

  ```cpp
  #include <algorithm>
  long long solution(int n, vector<int> works) {
    long long answer = 0;

    // 1번 : 내림차순 정렬
    sort(works.begin(),works.end(),greater<int>());

    // 2번
    long long cnt = (-1 * n);
    for (int i = 0; i < works.size(); i++) cnt += works[i];
    if (cnt <= 0) return 0;

    // 3번 : 메인 로직
    int index = 1;
    while (n > 0){
      int MaxVal = works[index-1];
      int NextMaxVal = works[index];
        
      // 만약 앞뒤가 다르다면
      if(MaxVal != NextMaxVal){
        int tmpVal = MaxVal - NextMaxVal;

        // 3-1번 : n을 모두 사용해도 괜찮은지
        if (tmpVal * index <= n) {
          for (int i = 0; i < index; i++) works[i] -= tmpVal;
          n -= (tmpVal * index);
        }
        else {
          int i = 0;
          while (n > 0) {
            if(i >= index) i = 0;
            works[i++]--;
            n--;
          }
        }
      }
      // 3-2번
      else if (MaxVal == NextMaxVal && index + 1 >= works.size()) {
        int i = 0;
        while (n > 0) {
          if (i >= index) i = 0;
          works[i++]--;
          n--;
        }
      }
      index++;
    }  
    for (int i = 0; i < works.size(); i++) answer += pow(works[i],2);
    return answer;
  }
  ```

### 다른이의 풀이 (오답)

  > 이 방법도 생각했으나, 시간 복잡도를 초과할 것 같아서 사용하지 않았음. 역시나 시간 초과임.

  ```cpp
  #include <algorithm>  
  int solution(int n, vector<int> works){
    int answer = 0;

    while (n != 0){
      *max_element(works.begin(), works.end()) -= 1;
      n -= 1;
    }
    for (int i = 0; i < works.size(); i++) answer += works[i] * works[i];
    return answer;
  }
  ```

  > 이놈도 시간 초과임

  ```cpp
  #include <numeric>
  int solution(int n,vector<int> works){
    int answer = 0;
    int sum = std::accumulate(works.begin(), works.end(), 0);
    if (n < sum){
      for (int i = 0; i < n; i++){
        int maxIndex = find(works.begin(), works.end(), *max_element(works.begin(), works.end())) - works.begin();
        works[maxIndex] -= 1;
      }

      int worksSize = works.size();
      for (int j = 0; j < worksSize; j++) answer += pow(works[j], 2);
    }
    return answer;
  }
  ```
