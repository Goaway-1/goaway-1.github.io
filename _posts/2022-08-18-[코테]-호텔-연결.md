## 문제

  철수는 엘리트 나라에서 발달이 잘 안된 외진 곳에 N개의 호텔들을 건설했다. N(1≤N≤1,000)개의 호텔은 (편의상 1...N) 평면(0≤X≤1,000,000; 0≤Y≤1,000,000)상의 정수 좌표로 위치를 나타낸다. 대부분의 호텔은 서로 연결되는 도로가 존재하지 않는다. 하지만 몇 개의 호텔은 서로 이미 도로가 연결되어 있는데, 이미 존재하는 두 개의 호텔을 연결하는 M 개의 도로(1≤M≤1,000)가 주어진다. 철수는 그의 모든 호텔을 연결하는 추가적인 도로를 건설하려고 한다. 추가적으로 건설되는 __도로의 길이가 최소가 되도록 여러분이 도와주세요.__

## 조건

  - 호텔의 개수 N과 연결된 호텔들의 정보 M개가 주어진다.
  - 1번 호텔부터 N번 호텔까지의 좌표(x,y)가 입력되어 있는 arr 배열과 이미 연결되어 있는 호텔들의 정보가 bridge 배열에 담겨있다.

## 입출력 예시

  |N|M|arr|bridge|result|
  |:--:|:--:|:--:|:--:|:--:|
  |5|2|[[2,1],[4,1],[5,1],[4,3],[5,1],[1,2]]|[[1,2],[2,3]]|3.41|
  
## 풀이방식

  > Union & Find() 메서드를 사용하여 서로 연결되어 있는지 확인하고, 최단 거리를 연결

  1. Union&Find를 사용하여 연결되어 있는 호텔을 확인
  2. 호텔간 연결 가능한 다리를 hot배열에 저장
  3. 거리가 가장 짧은 순서대로 정렬 후, 연결
      - 연결은 Union을 의미

  ```cpp
  // 호텔간의 연결을 표시하기 위한 배열
  int ch[1001];
  
  // 호텔간의 거리를 나타내는 구조체
  struct hotel{
    int x, y;   // 호텔 번호
    float val;  //거리
    
    hotel(int a,int b,float c) : x(a), y(b), val(c) {};
    
    bool operator < (const hotel &b) const{
      return val < b.val;
    }
  };
  int Find(int v){
    if(v == ch[v]) return v;
    else return Find(ch[v]);
  }
  void Union(int a,int b){
    a = Find(a);
    b = Find(b);
    if(a != b) ch[a] = b;
  }

  float solution(int N,int M,vector<pair<int,int>> arr,vector<pair<int,int>> bridge){	
    float answer = 0.f; 
    vector<hotel> hot;		// 호텔간 연결할 수 있는 다리 모음 
    
    // INIT Ch
    for(int i=1;i<=n;i++)ch[i] = i; 

    // 1. bridge 배열을 통해 연결되어 있는 호텔을 확인 | Union & Find
    for(int i = 0;i<bridge.size();i++){
      Union(bridge[i].first, bridge[i].second);
    }
    
    // 2. 호텔간 연결 가능한 다리를 vector에 모아둠 | i와 j는 호텔번호 
    for(int i=1;i<N;i++){
      for(int j=i+1;j<=N;j++){
        float tmp = sqrt(pow((arr[i].first - arr[j].first),2) + pow((arr[i].second - arr[j].second),2));
        hot.push_back(hotel(i,j,tmp));
      }
    }
    
    // 3. 거리가 가장 짧은 순서대로 정렬 후 합산
    sort(hot.begin(),hot.end());
    for(int i=0;i<hot.size();i++){
      if(Find(hot[i].x) != Find(hot[i].y)){			
        answer += hot[i].val;
        Union(hot[i].x,hot[i].y);
      }
    }
    
    return ceil(answer * 100) / 100;
  }
  ```


## 개선점
  - Union & Find개념을 자꾸 잊어버리는데 별거 없다. 
    - 배열에 정보들을 저장하고, Find를 통해 값을 찾아 다르면 Union하면 된다.
