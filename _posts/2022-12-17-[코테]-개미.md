## 문제

  개미가 일렬로 이동할 때, 가장 앞의 개미를 제외한 나머지 개미는 모두 앞에 개미가 한 마리씩 있다. 

  서로 반대 방향으로 이동하던 두 개미 그룹이 좁은 길에서 만났을 때, 개미는 어떻게 지나갈까?

  최근 연구에 의하면 위와 같은 상황이 벌어지면 개미는 서로를 점프해서 넘어간다고 한다.

  즉, 두 그룹이 만났을 때, 1초에 한번씩 개미는 서로를 뛰어 넘는다. (한 개미가 다른 개미를 뛰어 넘고, 다른 개미는 그냥 전진한다고 생각해도 된다)

  하지만 모든 개미가 점프를 하는 것은 아니다. 자신의 앞에 반대 방향으로 움직이던 개미가 있는 경우에만 점프를 하게 된다.

  첫 번째 그룹이 ABC로 움직이고, 두 번째 그룹의 개미가 DEF순으로 움직인다고 하자. 그럼, 좁은 길에서 만났을 때, 개미의 순서는 CBADEF가 된다. 1초가 지났을 때는 자신의 앞에 반대방향으로 움직이는 개미가 있는 개미는 A와 D다. 따라서, 개미의 순서는 CBDAEF가 된다. 2초가 되었을 때, 자신의 앞에 반대 방향으로 움직이는 개미는 B,D,A,E가 있다. 따라서, 개미의 순서는 CDBEAF가 된다.

  T초가 지난 후에 개미의 순서를 구하는 프로그램을 작성하시오.


  [링크](https://www.acmicpc.net/problem/3048)
  
## 조건

  - 첫 번째 줄에 첫 번째 그룹의 개미의 수 N1과 두 번째 그룹의 개미의 수 N2가 주어진다.
  - 다음 두 개 줄에는 첫 번째 그룹과 두 번째 그룹의 개미의 순서가 주어진다. 각 개미는 알파벳 대문자로 표현할 수 있으며, 두 그룹에서 중복되는 알파벳은 없다.
  - 마지막 줄에는 T가 주어진다. (0 ≤ T ≤ 50)
  
## 입출력 예시
  
  |n1|n2|ant1|ant2|t|result|
  |:--:|:--:|:--:|:--:|:--:|:--:|
  |3|3|ABC|DEF|0|CBADEF|
  |3|3|ABC|DEF|2|CDBEAF|
  |3|4|JLA|CRUO|3|CARLUJO|

## 풀이방식

### 나의 풀이방식

  > 시뮬레이션으로 자세한 풀이는 아래...

  1. 두 개미 그룹이 만났을때를 하나의 String에 저장
  2. 각 오른/왼쪽으로 가는 개미의 선두 인덱스를 rt/lt에 저장
      - 이때 rt/ltEnd는 선두가 끝에 도달했음을 의미
  3. 이제 T번 반복하며, 위치를 바꿔주며 초기 1번째는 rt와 lt를 바꿔준다.
      - 그 외에는 각 rt/lt의 전방에 있는 좌표와 바꿔준다. (rt + 1, lt - 1)
          - 단 이때, rt/lt의 전방에 아무것도 없다면 도달했다는 뜻임으로, 선뒤를 따라 멈처있는 가장 뒤 개미를 인덱스로 바꿔준다.
      - RT와 LT사이에 
  4. string을 출력하고 종료한다.

  ```cpp
  #include <iostream>
  #include <vector>
  #include <string>
  #include <algorithm>
  using namespace std;

  // 두 인덱스의 값을 바꾸어줌
  void swaps(int idx1, int idx2, string& ans) {
    int tmp = ans[idx1];
    ans[idx1] = ans[idx2];
    ans[idx2] = tmp;
  }

  int main(void) {
    int n1 = 0, n2 = 0, t = 0;
    string ant1 = "", ant2 = "", answer = "";
    cin >> n1 >> n2 >> ant1 >> ant2 >> t;

    // 1번 : 초기 만났을때의 배열
    for (int i = ant1.size() - 1; i >= 0; i--) answer += ant1[i];
    answer += ant2;

    /* 2번 : lt는 좌측, rt는 우측으로 이동하는 가장 선두의 좌표를 의미
    *  rt/ltEnd는 각 선두가 끝에 도달했음을 의미 */
    int rt = ant1.size() - 1, lt = ant1.size();
    bool rtEnd = false, ltEnd = false;

    // T번 반복
    for (int i = 0; i < t; i++) {
      if (i == 0) swaps(lt, rt, answer);
      else {
        // RT 전방과 교체
        if (!rtEnd && rt + 1 < answer.size()) swaps(rt, rt + 1, answer);
        else if (rtEnd) rt--;

        // LT 전방과 교체
        if (!ltEnd && lt - 1 >= 0) swaps(lt, lt - 1, answer);
        else if (ltEnd) lt++;

        // RT와 LT 사이에서 교체가 가능하다면...
        if (rt - lt > 2) {
          for (int j = lt + 1; j < rt; j += 2) {
            swaps(j, j + 1, answer);
          }
        }
      }

      // RT/LT의 도달 여부
      if (!ltEnd && lt - 1 >= 0) lt--;
      else if (!ltEnd) { ltEnd = true; }
      if (!rtEnd && rt + 1 < answer.size()) rt++;
      else if (!rtEnd) { rtEnd = true; }
    }

    cout << answer;
    return 0;
  }
  ```

### 다른이의 풀이

  > S1과 S2 두개로 나누고, 단순히 S1[i], S2[i + 1]를 스왑...

  ```cpp
  #include <iostream>
  #include <string>
  #include <algorithm>

  using namespace std;

  int N1, N2, T;
  string s1, s2;
  void input() {
    cin >> N1 >> N2;
    cin >> s1;
    cin >> s2;
    cin >> T;
  }

  void solve() {
    reverse(s1.begin(), s1.end());
    string s = s1 + s2;
    while (T--) {
      for (int i = 0; i < N1 + N2 - 1; i++) {
        if (s1.find(s[i]) != string::npos && s2.find(s[i + 1]) != string::npos) {
          swap(s[i], s[i + 1]);
          i++;
        }
      }
    }
    cout << s;
  }
  int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    cout.tie(nullptr);

    input();
    solve();
  }
  ```


## 개선점
  - 그냥 입력받고, 스왑하면 되는걸... 앞의 개미는 순서 바꾸고 멍청하게 풀었다..
