## 항목 1 : c++를 언어들의 연합체로 바라보자.

  c++은 __"C, 객체지향의 C++, 템플릿의 C++, STL"과__ 같이 4가지 언어들의 기능을 모두 포함하고 있는데, 이 언어 중 어떤 기능을 사용하냐에 따라 프로그램의 규칙이 달라진다.
  
  만약 예를들어 "값 전달"과 "참조 전달" 중 한가지 전달방법을 선택할때 언어에 따라 효과적인 전달방법이 있으며 아래와 같다.

  |기능|전달|이유|
  |:--:|:--:|:--:|
  |C|값 전달|형식적인 이유|
  |객체지향의 C++|참조 전달|생성자/소멸자 개념이 생겼기에|
  |템플릿의 C++|참조 전달|객체 타입조차 알 수 없기에|
  |STL|값 전달|C의 포인터를 본떠 만든 것이기에|

### 잊지 말자

  - C++은 경우에 따라 효과적인 프로그래밍 규칙이 달라진다. 이때 경우는 위에서 설명하는 4가지 언어를 의미한다.

## 항목 2 : #define을 쓰려거든 const, enum, inline을 떠올리자.

### __매크로__

  ```c++
  #define A 1.6
  ```
  
  매크로(선행처리자)로 위에서 선언된 A는 "기호식 이름"처럼 보이지만, 컴파일러에게 넘어가기 전에 선행처리자가 상수로 변환하기 때문에 A라는 이름은 "기호 테이블"에 들어가지 않는다.
  
  __문제점__ : 이는 에러발생 시, A는 "기호식 이름"으로 표시되지 않고 "상수"로 표시되기 때문에 혼란을 야기한다. 특히 다른 개발자가 매크로를 선언했을때는 오류를 찾을 수가 없다. 또한 선행처리자가 A를 찾아 모두 상수로 바꾸기 때문에 메모리 사용측면에서 비효율적이다. 

###  __※ 해결방법__
      
  ```c++
  const double A = 1.6;
  ```

  이를 해결하기 위해서는 매크로 대신 __"상수"를__ 사용하는 것이다. 이는 기호 테이블에 들어가기 때문에 컴파일러에 표시되며, 여러번 쓰여도 사본은 1개만 생성되기 때문에 메모리 점유도 효율적이다.

### __※ 주의사항__

  1. 상수  포인터를 정의하는 경우
      ```c++
      const char* const A = "~~";   //구시대적
      const string A("~~");
      ```

      - 위와 같이 포인터와 포인터가 가르키는 대상까지 const로 선언해야 되며, C언어의 구시대적 char*보다는 string 방식을 쓰는 것을 추천한다.

  2. 클래스 멤버로 상수를 정의하는 경우
    
      이는 즉 클래스 상수를 공유하는 "정적(Static)"멤버를 만들때를 의미하며, 이때 해결 방법은 2가지가 존재한다.

      > 1번 방안
      ```c++
      //header
      class A{
      private:
        static const int tmp = 5;   //정의가 아닌 선언
        int score[tmp];
      };
      //cpp
      const int A::tmp;             //클래스 상수의 주소를 요구할때
      ```
      이때 tmp는 선언시 초기화되며, 클래스 상수의 주소를 요구할 시, cpp에 정의해준다.

      하지만 구 컴파일러의 경우 선언시점에 초기값 선언이 옳지 않다고 판단하게 될 수 있는데, 이는 아래와 같이 초기값을 정의 시점으로 옮겨서 해결한다.

      ```c++
      //header
      class A{
      private:
        static const int tmp;           //정의
      };
      //cpp
      const int A::tmp = 5;             //선언
      ```

      > 2번 방안
      ```c++
      //header
      class A{
      private:
        enum {tmp = 5;}
        int score[tmp];
      };
      ```
      이는 만약 클래스 컴파일 도중 클래스의 상수값을 요구할때 __나열자(enum)을__ 사용하여 해결한다.

      이 나열자를 사용하면 수많은 장점이 존재하는데, 첫 번째로 선언한 정수 상수를 다른 개발자가 주소를 얻거나 참조하려할때 자물쇠역할을한다. 이를 통해서 접근을 제한한다.

      두 번째로 1번 방안에서 const는 컴파일러의 오류로 혹시 모르는 메모리가 할당될 가능성이 있지만 나열자는 그런 가능성이 존재하지 않는다.

      세 번재로 많은 개발자들이 이 방식을 사용하고 있으며, 이는 템플릿 메타프로그래밍의 핵심이다.

      때문에 enum을 사용하는 것을 추천한다.

### __매크로 함수__

  ```c++
  //매크로
  #define A(a,b) f((a) > (b) ? (a) : (b)

  //오류
  A(++a,b);
  ```

  위와 같이 매크로 함수가 존재할때, ++a는 b와의 관계에 따라 2번 증가하거나 1번 증가하는 괴현상이 발생한다.

### __※ 해결방법__
  
  ```c++
  template <typename T>
  inline void A(const T &a, const T &b){
    f(a > b ? a : b)
  }
  ```

  위와같이 인라인함수에 대한 템플릿으로 구현한다. 이는 진짜 함수이기 때문에 유효범위와 접근규칙을 준수한다.

### 잊지 말자

  - 단순 상수는 #define이 아닌 "const 객체","enum"을 우선 사용하고, 매크로 함수는 "인라인 함수"를 사용한다.
